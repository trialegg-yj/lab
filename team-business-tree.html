🥊 프로젝트 DB — 모든 업무의 창고 가장 중요
팀의 모든 업무가 담기는 유일한 DB예요. 예전에는 업무함·주간업무보고가 따로 있었지만, 이제는 전부 여기 하나로 합쳐졌어요.
큰 방향(프로젝트)부터 세부 할 일(태스크)까지 같은 곳에서 관리해요.
🏗️ 3단계 계층 구조 (레벨) 구조
프로젝트 DB 안의 항목은 세 단계로 나뉘어요. 큰 것이 위, 세부적인 것이 아래예요.
🏆프로젝트 — 팀이 달성하려는 큰 목표·방향 최상위
📦에픽 — 목표를 이루기 위한 솔루션/전략 단위 중간
✅태스크 — 각 에픽을 실행하는 실제 액션 실행
예시 1. 룻 메뉴 변경 (프로젝트)
   → 26년 봄 (에픽)
      → 26년 봄 기획  ·  26년 봄 레시피 개발  ·  원가 계산 (태스크)
예시 2. 화장품 책임판매업 등록 (프로젝트)
   → 서류 준비  /  기관 제출·신청  /  후속 대응 (에픽)
      → 화장품책임판매업 등록 신청서 작성  ·  제출 기관 확인  ·  화장품 책임판매관리자 교육 이수 (태스크)

03 · 사용법
처음 써보는 분을 위한 단계별 가이드
순서대로 따라 하면 바로 쓸 수 있어요.
1

대시보드 열기

노션에서 Team Business 대시보드를 열어요. 이 페이지가 모든 것의 시작이에요.

2

내 할 일 확인하기

🥊 프로젝트 섹션에서 "내 담당" 뷰를 클릭하면 내가 담당자로 지정된 항목만 보여요. 매일 아침 이 뷰로 하루를 시작하세요.
(내 개인 페이지에서 시작할 수도 있어요. 개인 페이지 안에서 필터·정렬 등 보기 뷰는 자유롭게 원하는 대로 바꿔가며 사용할 수 있어요.)

3

새 할 일 만들기

+ 새로 만들기를 눌러 항목을 추가해요.

자동으로 채워지는 값 (생성 후 변경 가능)
→ 담당자 · 시작일 · 진행상태 (기본값: 시작 전)

직접 입력해야 하는 값
→ 레벨 (🏆 프로젝트 / 📦 에픽 / ✅ 태스크 중 선택)
→ 분야 (사업기획 / 사업운영 / 주류사업 등)
→ 우선순위 (P1~P4 중 선택)
→ 마감일

4

진행상태를 기록해요

진행률은 체크박스 기준으로 집계돼요. 완료된 태스크는 반드시 체크를 표시해서 완료 처리해주세요.
세부 진행상태는 상태값(진행 중 / 보류 / 완료 등)으로 표시하면 돼요.

5

미팅 후 회의록은 공유일정 안에 남겨요

팀 전체가 알아야 하는 일정은 📅 공유일정에 추가해요.
회의 시작 전 녹음 버튼을 눌러 진행하고, 미팅이 끝나면 완료 여부를 체크해주세요.

6

주간회의 기록 남기기

매주 월요일 오전 9시 30분에 주간회의록이 해당 날짜 공유일정 페이지에 자동으로 생성돼요.
주요 논의 내용과 액션 아이템을 기록해요.

💡 헷갈릴 때 이것만 기억하세요
업무는 → 🥊 프로젝트 DB  ·  일정/회의는 → 📅 공유일정
04 · 기능 가이드
상황별로 이렇게 써요
내 상황과 가장 가까운 케이스를 선택하세요.
"룻 메뉴 변경" 프로젝트를 팀에서 새로 시작하기로 했어요.
큰 방향(프로젝트) → 작업 묶음(에픽) → 세부 액션(태스크) 순서로 만들면 돼요.
1

대시보드 🥊 프로젝트 섹션에서 프로젝트 추가하기 클릭
레벨이 🏆 프로젝트로 고정된 페이지가 생성돼요. [직접 진행] 업무명 템플릿이 자동 적용돼요.

2

페이지 열어서 제목·분야·우선순위·마감일 입력
담당자·시작일·진행상태는 자동으로 채워져 있어요. 필요하면 변경할 수 있어요.

3

에픽 추가하기 클릭 → "26년 봄" 에픽 생성
상위 항목에 방금 만든 프로젝트를 연결하면 진행률이 자동 집계돼요.

4

태스크 추가하기(직접 진행) 클릭 → "26년 봄 기획" 등 세부 태스크 생성
상위 항목에 에픽을 연결하면 에픽 진행률에도 반영돼요.

5

태스크가 끝나면 완료여부 체크박스 클릭 → 상위 에픽·프로젝트 진행률이 자동으로 올라가요.

내가 직접 하기 어려운 업무를 팀원에게 넘겨야 해요.
요청 태스크는 제약사항·맥락을 작성하는 섹션이 포함된 별도 템플릿으로 만들어져요.
1

태스크 추가하기(요청) 클릭
레벨이 ✅ 태스크로 고정된 페이지가 생성돼요. [요청] 업무명 템플릿에는 직접 진행과 달리 제약사항 섹션이 포함돼요.

2

담당자에 요청할 팀원 지정, 참조자에 나를 추가하면 진행 상황을 계속 공유받을 수 있어요.

3

템플릿의 제약사항 섹션에 배경·요청 범위·마감 이유를 적어두면 받는 사람이 맥락을 빠르게 파악할 수 있어요.

4

상위 항목에 관련 에픽·프로젝트 연결 → 어느 프로젝트의 일인지 연결고리가 생겨 나중에 추적하기 쉬워요.

출근하자마자 오늘 뭐부터 해야 할지 정리하고 싶어요.
대시보드 임베딩 뷰, 프로젝트 DB, 개인 페이지 세 곳에서 확인할 수 있어요.
1

대시보드 🥊 프로젝트 섹션 → 내 프로젝트 탭 클릭
내가 담당자인 항목만 즉시 확인할 수 있어요.

2

분야별 탭으로 전환하면 사업기획·주류사업 등 업무 카테고리 기준으로 묶어서 볼 수 있어요.

3

더 상세하게 보려면 프로젝트 바로가기 클릭 → 내 담당 뷰
우선순위(P1→P4)·마감일 기준으로 자동 정렬된 내 전체 업무가 한눈에 보여요.

4

개인 페이지에서는 필터·정렬을 자유롭게 바꿔가며 나만의 뷰로 커스텀할 수 있어요.

다음 주 외부 미팅이 잡혔어요. 팀에 공유하고 나중에 회의록도 남기고 싶어요.
모든 팀 일정과 회의록은 📅 공유일정에서 통합 관리해요.
1

대시보드 📅 공유일정 섹션 → 일정 추가 버튼 클릭
날짜·참여자·태그(외부미팅)·장소 입력 후 저장하면 팀 전체에 공유돼요.

2

달력 뷰로 전환하면 팀 전체 일정을 월·주 단위로 한눈에 확인할 수 있어요.

3

미팅 당일, 회의 시작 전 녹음 버튼을 눌러 진행해요.

4

회의가 끝나면 해당 공유일정 페이지 안에 회의록을 기록하고 완료여부 체크
"11일 미팅 회의록 어디 있지?" → 11일 공유일정 페이지 안을 확인하면 돼요.

5

매주 월요일 오전 9시 30분, 주간회의록이 공유일정에 자동 생성돼요. 안건·결정사항을 채워넣으면 돼요.

이번 분기 프로젝트들이 어떻게 돌아가고 있는지 한눈에 보고 싶어요.
프로젝트 DB의 다양한 뷰를 목적에 맞게 골라 쓰면 돼요.
1

대시보드 타임라인 탭 → 시작일 기준 프로젝트·에픽 일정 흐름을 시각적으로 파악해요.

2

프로젝트 DB → 프로젝트 현황 뷰 → 프로젝트별 진행률 한눈에 확인

3

칸반보드 뷰 → 진행상태(시작전·진행중·완료·보류) 컬럼별로 프로젝트 현황을 시각화해요.

4

담당자별 뷰 → 팀원별로 어떤 업무가 얼마나 있는지 확인해요.

5

전체(빈값확인) 뷰 → 우선순위·담당자·시작일·마감일 중 하나라도 비어있는 항목이 표시돼요.
주기적으로 확인해서 빈값을 채워주세요.

마케팅 관련 좋은 아티클을 발견했어요. 팀에 공유하고 나중에 다시 찾기도 쉽게 하고 싶어요.
🔍 공유자료실에 올려두면 팀 전체가 분야별·유형별로 언제든 찾을 수 있어요.
1

공유자료실 섹션 → 자료 추가 버튼 클릭
파일/링크·한줄요약·분야·자료유형·사업부를 입력해요. 한줄요약을 잘 써두면 나중에 찾기 훨씬 편해요.

2

자료를 찾을 때는 목적에 맞는 뷰를 골라요
사업부별 BD·LB·Tech 기준  ·  분야별 마케팅·영업 등  ·  자료유형별 아티클·레퍼런스·영상 등

3

자주 참고하는 자료는 즐겨찾기에 나를 추가 → 내 즐겨찾기 뷰에서 바로 접근할 수 있어요.
